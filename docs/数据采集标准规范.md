# 临床多模态情绪数据采集标准规范

## Clinical Multimodal Emotion Data Collection Standards

> **目的:** 为医院后续采集乳腺癌患者多模态情绪数据提供标准化指南，确保数据可用于训练"情绪识别→风险评估→自适应干预"全链路AI系统。
> **版本:** v1.0 | **日期:** 2026-06-15

---

## 一、采集模态清单 (Modalities to Collect)

### 1.1 必须采集 (Required)

| 模态 | 设备 | 采样规格 | 时长 | 说明 |
|------|------|---------|------|------|
| **🎤 语音** | 麦克风 (建议采样率≥16kHz) | 16kHz, 16bit, mono WAV | ≥30秒/次 | 患者自然对话或标准化朗读 |
| **📷 面部视频** | 摄像头 (≥1080p, ≥25fps) | 1080p, 25-30fps, MP4 | ≥30秒/次 | 正面面部，光照均匀 |
| **📋 自评量表** | 平板/手机问卷 | JSON结构化存储 | 每次采集同步 | PHQ-9, GAD-7, 疼痛NRS等 |

### 1.2 强烈建议采集 (Strongly Recommended)

| 模态 | 设备 | 采样规格 | 说明 |
|------|------|---------|------|
| **💓 心率/HRV** | PPG手环/胸带 | ≥25Hz, ≥5分钟窗口 | 自主神经活动指标 |
| **🛏️ 睡眠质量** | 可穿戴/自评 | 每日 | 入睡时间、总时长、深睡比例 |
| **🚶 活动量** | 加速度计/手机 | 每日步数/活动分钟数 | 身体功能维度 |
| **📝 自由文本** | 语音转写/打字 | UTF-8文本 | 主观感受描述（自然语言） |

### 1.3 长期可选 (Optional, Long-term)

| 模态 | 设备 | 说明 |
|------|------|------|
| **🧠 EEG** | 便携脑电帽 | 情绪神经相关物，科研级 |
| **🌡️ 皮肤电(GSR)** | Empatica/Shimmer | 情绪唤醒度指标 |
| **👁️ 眼动追踪** | 屏幕式/眼镜式眼动仪 | 注意力/回避模式分析 |

## 二、情绪标注标准 (Emotion Annotation Protocol)

### 2.1 核心标注维度

```
基于 Russell 的 Circumplex Model 进行二维连续标注：

        高唤醒 (High Arousal)
              |
    紧张 ──── 兴奋
    愤怒 ──── 高兴
              |
  低愉悦 ────┼──── 高愉悦 (High Valence)
              |
    悲伤 ──── 放松
    抑郁 ──── 平静
              |
        低唤醒 (Low Arousal)
```

### 2.2 标注量表

每一次采集进行以下标注（1-9分或0-100分）：

| 标注项 | 中文名称 | 量表范围 | 说明 |
|--------|---------|---------|------|
| **Valence** | 愉悦度 | 1-9 (1=极不愉悦, 9=极愉悦) | 情绪的正负性 |
| **Arousal** | 唤醒度 | 1-9 (1=极度平静, 9=极度兴奋) | 情绪的强度 |
| **Depression** | 抑郁程度 | 0-100 (PHQ-9总分的映射) | 核心目标变量 |
| **Anxiety** | 焦虑程度 | 0-100 (GAD-7总分的映射) | 次要目标变量 |
| **Fatigue** | 疲劳程度 | 0-10 (NRS) | 癌症相关疲劳 |
| **Pain** | 疼痛程度 | 0-10 (NRS) | 疼痛评分 |

### 2.3 标注流程 (Protocol)

```
第〇步: 采集前环境准备
  ├── 安静、光线均匀的房间
  ├── 摄像头高度与眼睛平齐，距离约0.6-0.8m
  ├── 麦克风距口部约20-30cm
  └── 背景噪声 < 45dB

第一步: 标准化引导 (2分钟)
  ├── 告知患者采集目的和流程
  ├── 签署知情同意书
  └── 简要说明评分量表含义

第二步: 视频+音频同步采集 (3-5分钟)
  ├── 标准化访谈问题：
  │   ├── "请描述您今天感觉怎么样？" (开放式)
  │   ├── "您现在的情绪是怎样的？" (引导情绪表达)
  │   ├── "最近的治疗给您带来了哪些困扰？" (症状探查)
  │   └── "有什么事情让您感到开心或担忧？" (正负情绪触发)
  └── 鼓励患者自然表达，不打断

第三步: 患者自评 (2分钟)
  ├── 完成 PHQ-9 抑郁筛查量表
  ├── 完成 GAD-7 焦虑筛查量表
  ├── 完成 NRS 疼痛和疲劳评分
  └── 自由文本：写下此刻最想说的话

第四步: 专业人员标注 (后台)
  ├── 2名经过培训的标注员独立标注
  ├── Valence + Arousal 根据视频+音频独立评分
  ├── 标注一致性检验 (ICC > 0.7)
  └── 分歧时第3名标注员裁决
```

## 三、标准化采集设备规范

### 3.1 摄像头要求

| 参数 | 最低要求 | 推荐要求 |
|------|---------|---------|
| 分辨率 | 1920×1080 | 1920×1080 |
| 帧率 | 25 fps | 30 fps |
| 编码 | H.264 | H.264/H.265 |
| 对焦 | 自动对焦，面部优先 | 定焦，固定距离 |
| 光照补偿 | 自动 | 手动固定色温(6500K) |

### 3.2 麦克风要求

| 参数 | 最低要求 | 推荐要求 |
|------|---------|---------|
| 采样率 | 16000 Hz | 48000 Hz |
| 位深 | 16 bit | 24 bit |
| 声道 | 单声道(Mono) | 单声道 |
| 格式 | PCM WAV | PCM WAV |
| 信噪比 | >60 dB | >80 dB |

### 3.3 采集设备方案对比

| 方案 | 设备 | 成本 | 部署难度 | 适用场景 |
|------|------|------|---------|---------|
| **方案A-基础** | 笔记本内置摄像头+麦克风 | ¥0 | ★☆☆☆☆ | 快速试点 |
| **方案B-标准** | Logitech C920 + Blue Yeti | ¥1,500 | ★★☆☆☆ | 科室固定采集 |
| **方案C-专业** | 工业相机 + 领夹麦 + 补光灯 | ¥5,000 | ★★★☆☆ | 科研级数据 |
| **方案D-移动** | iPad/平板前置 + APP | ¥3,000 | ★★☆☆☆ | 床旁移动采集 |

> **建议:** 先用方案A或B启动小规模试点(30-50例)，验证流程后扩展。

## 四、采集时间节点 (Time Points)

### 4.1 乳腺癌化疗患者典型采集节点

```
治疗周期示意图：

  [基线] ──→ [化疗1] ──→ [化疗2] ──→ ... ──→ [化疗N] ──→ [术后] ──→ [随访]
    ↓          ↓          ↓                    ↓          ↓          ↓
   采集1      采集2      采集3                采集N      采集N+1    采集N+2

每个化疗周期内建议采集2-3次：
  ├── T0: 化疗前1天 (基线情绪/症状)
  ├── T1: 化疗后第3天 (副作用高峰期)
  └── T2: 化疗后第7天 (恢复期)
```

### 4.2 推荐采集频次

| 数据类型 | 频次 | 说明 |
|---------|------|------|
| 视频+音频情绪采集 | 每周期2-3次 | 标准化访谈录制 |
| 自评量表(PHQ-9/GAD-7) | 每周期2-3次 | 与视频采集同步 |
| 疼痛/疲劳评分(NRS) | 每日1次 | 可通过手机APP采集 |
| 可穿戴生理数据 | 连续监测 | 24h心率/活动量 |
| 睡眠质量 | 每日1次(晨起) | 自评或可穿戴自动 |

## 五、数据存储格式标准

### 5.1 文件命名规范

```
格式: {患者ID}_{时间点}_{模态}.{扩展名}

示例:
  P001_Baseline_video.mp4        ← 患者001，基线，视频
  P001_Baseline_audio.wav        ← 患者001，基线，音频
  P001_C1D3_video.mp4            ← 患者001，化疗第1周期第3天，视频
  P001_C1D3_selfreport.json      ← 患者001，自评量表
```

### 5.2 目录结构

```
采集数据根目录/
├── P001/
│   ├── P001_Baseline_video.mp4
│   ├── P001_Baseline_audio.wav
│   ├── P001_Baseline_selfreport.json
│   ├── P001_C1D3_video.mp4
│   ├── P001_C1D3_audio.wav
│   ├── P001_C1D3_selfreport.json
│   ├── P001_C1D7_video.mp4
```

## 七、伦理与隐私要求

## 八、采集质量检查清单 (QA Checklist)

### 每次采集后检查

**音频质量:**
- [ ] 采样率 = 16000Hz ✅ / 否则拒收 ❌
- [ ] 无削波/过载
- [ ] 信噪比 > 30dB
- [ ] 无明显环境噪声（空调、人声重叠）
- [ ] 语音占比 > 50%（非静音段）

**视频质量:**
- [ ] 分辨率 ≥ 1920×1080
- [ ] 帧率 ≥ 25fps
- [ ] 面部可见且光照均匀
- [ ] 无过度模糊/运动模糊
- [ ] 面部未遮挡（口罩、墨镜等 — 但有口罩数据也可保留作子集）

**标注质量:**
- [ ] PHQ-9/GAD-7所有条目完成
- [ ] 标注员间ICC(Intraclass Correlation) > 0.7
- [ ] 无缺失关键标注字段

## 九、最小可行采集方案（快速启动建议）

如果医院资源有限，**最精简方案**只需：

```
设备:    一台iPad + 一个安静房间
每次耗时:  5分钟
采集内容:
  1. iPad前置摄像头录制 3分钟标准化访谈 (视频+音频同步)
  2. iPad上完成 PHQ-9 + GAD-7 + 疼痛NRS (2分钟)
标注:
  2名护士/研究人员独立观看视频标注 Valence + Arousal

这样就能得到模型训练的最小数据集:
  - Input: video + audio
  - Label: Valence, Arousal, Depression(PHQ-9), Anxiety(GAD-7)
  - Clinical: Pain, Fatigue, Treatment Phase
```

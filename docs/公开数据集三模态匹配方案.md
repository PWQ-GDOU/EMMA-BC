# 公开数据集匹配方案：量表 + 文本 + 语音/表情三模态数据规划

> **核心问题:** 是否存在同时包含「临床量表 + 心情日志文本 + 语音/表情视频」的公开数据集？  
> **短答案:** 没有完美匹配的数据集。但可以通过 **DAIC-WOZ + 多数据集组合 + 迁移学习策略** 来实现等效覆盖。
> **版本:** v1.0 | **日期:** 2026-06-15

---

## 一、医院需求的精确拆解

你需要的三类数据：

| 数据类型 | 具体内容 | 临床用途 |
|---------|---------|---------|
| **① 量表数据** | PHQ-9, GAD-7, NRS疼痛/疲劳等标准化量表 | 金标准标签（Ground Truth） |
| **② 文本数据** | 患者手写/打字的今日心情日志（自由文本） | 主观感受的自然语言表达 |
| **③ 语音/表情数据** | 同步录制的面部视频 + 语音音频 | 客观行为信号（非言语情绪线索） |

**关键特征：这三类数据必须来自同一个患者的同一次采集**，才能做多模态对齐训练。

### 核心发现

**没有数据集满足全部三个要求。** 以下是原因：

- **DAIC-WOZ 的文本是访谈转写（transcript）**，不是患者主动写的心情日志。转写文本是"患者回答心理医生问题"，而非"我今天感觉…"的自由书写。
- **MODMA（中文数据集）有量表+语音，但没有视频和文本日志。**
- **社交媒体数据集（Reddit等）有量表+文本，但没有语音/视频。**

## 三、最接近的数据集详解

### 🥇 DAIC-WOZ — 榜首候选（最接近医院需求）

| 属性 | 详情 |
|------|------|
| 全称 | Distress Analysis Interview Corpus - Wizard of Oz |
| 来源 | University of Southern California (USC) |
| 参与人数 | 189人（含抑郁症患者和对照） |
| 数据内容 | 每位参与者一段 7-33 分钟的临床访谈 |

**包含的三模态：**

```
① 量表数据:
   ├── PHQ-8 (Patient Health Questionnaire, 0-24分)
   ├── GAD-7 部分条目
   ├── PCL-C (PTSD Checklist)
   └── 人口学信息 (年龄、性别、种族等)

② "文本"数据:
   ├── 访谈完整转写文本 (ASR + 人工校对)
   ├── ⚠️ 实质是对话文本，不是心情日志
   └── 格式: "Ellie: How are you feeling today? → Participant: I've been..."

③ 语音/表情数据:
   ├── 视频: 1080p 面部录像 (H.264)
   ├── 音频: 16kHz 单声道音频
   ├── 面部关键点: OpenFace提取的68个landmark + AU
   ├── 注视方向: 眼动追踪特征
   └── COVAREP音频特征: 74维声学特征
```

**与你医院需求的差距：**

| 差距项 | DAIC-WOZ现状 | 医院需求 | 差距程度 |
|--------|-------------|---------|---------|
| 文本类型 | 访谈对话转写 | 心情日志/日记 | 🔴 大 |
| 量表 | PHQ-8（缺第9项自杀意念） | PHQ-9完整版 | 🟡 中 |
| 人群 | 美国英语人群 | 中国乳腺癌患者 | 🔴 大 |
| 肿瘤特异性 | 无 | 化疗相关症状 | 🔴 大 |
| 采集环境 | 实验室访谈 | 医院自然场景 | 🟡 中 |

**获取方式:**
```
1. 访问: https://dcapswoz.ict.usc.edu/
2. 注册账号，填写学术使用申请
3. 签署数据使用协议 (DUA)
4. 等待审批 (通常1-4周)
5. 下载数据 (~50GB)
```

### 🥈 MODMA — 中文数据集（量表+语音）

| 属性 | 详情 |
|------|------|
| 全称 | Multi-modal Open Dataset for Mental-disorder Analysis |
| 来源 | 兰州大学 (Lanzhou University) |
| 参与人数 | 100+ 中国抑郁症患者 + 健康对照 |
| 语言 | 中文 |

**包含的数据：**

```
① 量表数据:
   ├── PHQ-9 ✅ (完整版，含自杀意念条目)
   ├── GAD-7 ✅
   ├── PSQI (匹兹堡睡眠质量指数)
   └── 临床诊断信息

② 文本数据:
   └── ❌ 无心情日志文本

③ 语音数据:
   ├── 语音录音 (结构化访谈 + 朗读 + 图片描述)
   ├── 128通道 EEG 数据
   └── ❌ 无面部视频
```

**优势:**
- ✅ 中文环境，与中国医院采集场景一致
- ✅ 完整 PHQ-9（含第9项）
- ✅ 结构化语音任务（访谈+朗读+图片描述）

**劣势:**
- ❌ 无面部表情视频
- ❌ 无自由文本心情日志
- ❌ 不是肿瘤患者

**获取方式:**
```
数据集论文: Cai et al., "MODMA dataset: A Multi-modal Open Dataset for 
           Mental-disorder Analysis", arXiv 2020
数据集网站: http://modma.lzu.edu.cn/
```

### 🥉 其他可补充的数据集

#### A. 纯文本心情日志数据集

| 数据集 | 文本类型 | 是否有量表 | 说明 |
|--------|---------|-----------|------|
| **eRisk** (CLEF) | Reddit 用户帖子 | ✅ 抑郁症自报标签 | 社交网络长文本，非日志 |
| **CLPsych** | 社交媒体帖子 | ✅ 临床评估标签 | Twitter/Facebook帖子 |
| **PsyQA** | 心理咨询对话 | ⚠️ 咨询师标注 | 中文心理咨询 |
| **Dreaddit** | Reddit 压力帖子 | ⚠️ 众包标注 | 10类压力标签 |
| **EMOTIVE** | 日常日记文本 | ⚠️ 情绪标签 | 日记风格文本 |

**⚠️ 关键限制：** 以上文本数据集都**没有**同步的语音/视频数据，无法做同一人的多模态对齐。

#### B. 纯语音/视频情绪数据集（无临床量表）

| 数据集 | 模态 | 样本量 | 用途 |
|--------|------|--------|------|
| RAVDESS | 音频+视频 | 24人×60段 | 情绪表达特征预训练 |
| CREMA-D | 音频+视频 | 91人×12段 | 多样化情绪表达 |
| AffectNet | 面部图像 | 45万张 | 面部表情识别预训练 |
| SEWA | 音频+视频 | 64人 | 自然对话场景情绪 |
| RECOLA | 音频+视频+ECG | 46人 | 协作任务情绪标注 |

## 四、推荐的三阶段数据策略

### 阶段 A: 用现有公开数据验证技术路线（1-2个月）

```
训练目标: 证明"多模态融合 > 单模态"的技术可行性

数据集组合:
  ┌─────────────────────────────────────────┐
  │  DAIC-WOZ (核心训练+验证)              │
  │  ├── 音频+视频+转写文本+PHQ-8          │
  │  └── 189人, 多模态对齐可用              │
  │                                         │
  │  RAVDESS + CREMA-D + AffectNet (预训练) │
  │  └── 语音/面部特征提取器预训练          │
  │                                         │
  │  MODMA (可选增强)                       │
  │  └── 中文语音+PHQ-9 → 跨语言验证       │
  └─────────────────────────────────────────┘

预期产出:
  ✅ MulT/AV-Transformer 音频+视频表情融合模型
  ✅ 情绪评分 (Valence + Arousal) 基线
  ✅ 抑郁程度回归 (PHQ-8 scores) 基线
  ✅ 多模态 vs 单模态性能对比
```

### 阶段 B: 为目标医院数据格式对齐构建Pipeline（1个月）

```
目标: 等医院有数据时能"即插即用"

工作内容:
  1. 编写"模拟医院数据生成器"
     └── 按照"数据采集标准规范"定义的格式
     └── 生成模拟的 量表JSON + 心情日志文本 + 模拟视频/音频

  2. 构建端到端数据处理Pipeline:
     ┌──────────────────────────────────┐
     │  [量表 JSON] ──→ [量表编码器]    │
     │  [日志文本]  ──→ [文本编码器]    │──→ [多模态融合] ──→ [输出]
     │  [音频 WAV]  ──→ [音频编码器]    │
     │  [视频 MP4]  ──→ [视频编码器]    │
     └──────────────────────────────────┘

  3. 验证Pipeline在模拟数据上跑通
     └── 确保医院真实数据到达后直接可用
```

### 阶段 C: 医院数据到达后 → 训练生产模型

```
数据采集 (医院):
  每位患者每次采集:
    ├── 3分钟标准化访谈视频
    ├── PHQ-9 + GAD-7 + NRS 量表 JSON
    └── 今日心情日志 (100-500字自由文本)

训练流程:
  1. 用公开数据预训练的模型作为初始权重
  2. 在医院数据上微调 (transfer learning)
  3. 持续学习 (每新增N个患者更新一次模型)
```

## 六、替代方案：如果 DAIC-WOZ 申请被拒

### 方案 1: 只用公开情绪数据集 + 模拟量表标签

```
1. 用 RAVDESS + CREMA-D 训练语音/视频特征提取器
2. 用 AffectNet 训练面部表情分类器
3. 用词向量/LLM 提取文本情感特征
4. 用 eRisk/CLPsych 数据训练文本-抑郁映射
5. 在医院自有数据上联合微调

风险: 没有多模态对齐的验证数据，系统集成风险高
```

### 方案 2: 与国内机构合作获取数据

```
可能的合作方:
  - 兰州大学 (MODMA 数据集团队)
  - 中科院心理所
  - 北京安定医院 / 上海精卫中心
  - 中山大学肿瘤防治中心

可合作获取: 中文临床抑郁检测数据 (可能含部分多模态数据)
```

### 方案 3: 合成数据增强

```
用现有数据 + 生成模型:
  1. 文本增强: 用 LLM 生成类似"心情日志"的文本
  2. 语音增强: 用 TTS + 情感控制生成不同情绪的语音
  3. 视频增强: 用面部动画生成对应情绪的视频

⚠️ 风险: 合成数据的分布可能与真实临床数据有偏差
```

```

我们将使用 DAIC-WOZ 数据集进行模型预训练和验证，探索音频、
视频和文本特征在抑郁程度评估中的融合方法。研究成果将仅用于
```

## 八、最终结论

```
问题: 有同时包含 量表+心情日志+语音/视频 的公开数据集吗？

回答: 没有。目前最接近的是 DAIC-WOZ (量表+访谈转写+视频+音频)
      和 MODMA (量表+语音，中文)，但都没有"心情日志"文本。

策略: 三段式方案
      A. DAIC-WOZ + RAVDESS + MODMA → 验证技术路线
      B. 预处理Pipeline就绪 → 等医院数据即插即用
      C. 医院数据到达 → 微调生产模型

医院独有价值: 心情日志文本 + 化疗周期标注 + 肿瘤特异性症状
              这些数据是公开数据集无法替代的，是项目的核心创新点
```
